Partie 1

Ce qui est déjà en place

Navigation globale — MenuWindow.tsx

  • Taskbar fixe Win95 persistante sur toutes les pages via transition:persist
  • 5 liens desktop + hamburger mobile avec dropdown animé (menu-slide-down)
  • État actif correct : shadow inversée desktop, fond bleu mobile
  • Détection intelligente : /projects/[slug] active l'item /catalogue
  • aria-label="Navigation" présent sur le bouton hamburger

Page Hero — index.astro

  • Structure en deux lignes : portrait + profil / 3 widgets statistiques
  • <h1 class="sr-only"> et description cachée pour les screen readers
  • Animations d'entrée window-bounce-in sur chaque fenêtre avec délais échelonnés
  • Badges de spécialisation staggerés (200ms + 60ms par item)
  • Badge de disponibilité avec pulse-glow pour attirer l'attention
  • Layout responsive : colonne sur mobile, 2 colonnes sur desktop
  • RoleWindow : rotation de rôles toutes les 1800ms + curseur clignotant animé

Page Mon Travail — main.astro

  • Sidebar avec catégories, compteurs et chevron indicateur
  • Cartes projets horizontales (image + titre + catégories + lien)
  • Hover : -translate-y-0.5 + shadow renforcée
  • Passage vers /catalogue avec filtre automatique via sessionStorage
  • Sidebar : scroll horizontal mobile, scroll vertical desktop

Catalogue — CatalogueFilter.tsx

  • Filtrage par catégorie single-select avec compteurs
  • Grille responsive (2 → 3 → 4 colonnes)
  • URL hash synchronisé (#project-id) — URL partageable
  • sessionStorage.wantedCat lu au mount pour auto-filtrage depuis /main
  • Modal plein écran pour le détail de chaque projet

Détail Projet — ProjectDetailPage.tsx

  • Hero image aspect-[16/7] en cover
  • Sections structurées : contexte, rôle, apprentissages, vision, compétences
  • Galerie en grille 2-3 colonnes avec lightbox intégré
  • Navigation prev/next dans le lightbox avec wrap circulaire
  • Animations d'entrée staggerées par section (150ms + 50ms par bloc)
  • Tags et lien externe vers le projet

Outils — outils.astro

  • Sidebar de 4 catégories (Web, Design, Audiovisuel, Bureautique)
  • 14 compétences avec progress bars animées au chargement
  • role="progressbar" sur chaque barre
  • Score numérique affiché (ex : "3.5/5"), icône par outil (36×36px)
  • CTA "Voir les projets →" en bas de sidebar

Contact — contact.astro + ContactForm.tsx

  • Formulaire 3 champs + validation client-side par champ
  • 4 états gérés : idle / loading / success / error
  • Auto-dismiss du message de succès (5 secondes)
  • Lien email fallback (maxime.mandin@icloud.com)
  • 3 fenêtres liens sociaux (Instagram, LinkedIn, Mon bureau) avec hover lift
  • Erreurs affichées sous chaque champ concerné

Système de design — global.css

  • Tokens cohérents : --color-win-surface, --color-win-title, --color-win-accent
  • Shadows bevel Win95 : --shadow-win-outset / --shadow-win-inset
  • 15+ animations CSS nommées et réutilisées dans tout le projet
  • @media (prefers-reduced-motion: reduce) respecté globalement
  • :focus-visible défini globalement (2px solid black avec offset)

SEO & accessibilité de base — BaseLayout.astro

  • Open Graph complet (og:title, og:image, og:url, og:locale)
  • Twitter Card configurée
  • JSON-LD : Person, LocalBusiness, VisualArtwork par projet
  • Sitemap auto-généré via @astrojs/sitemap
  • Vercel Analytics injecté dynamiquement
Partie 2

Ce qui reste à intégrer ou améliorer

UX Expérience utilisateur

Navigation & architecture de l'information

  • Pas de breadcrumb sur les pages /projects/[slug] — l'utilisateur ne sait pas où il se situe dans la hiérarchie
  • Pas de bouton "retour" dans le détail projet hors modal — seule la navigation globale existe
  • Les pages /main et /catalogue se chevauchent perceptivement sans différenciation claire dans la navigation
  • Pas de lien direct depuis la hero vers un projet spécifique — les fenêtres de la page d'accueil ne sont pas cliquables

Transitions & animations entre pages

  • Les View Transitions CSS (::view-transition-old/new) font référence à des keyframes slide-left / slide-right qui sont absentes de global.css — les transitions de page sont vraisemblablement silencieuses
  • Pas de transition de sortie sur les fenêtres lors d'un changement de page

Feedback utilisateur

  • Bouton submit du formulaire : l'état loading ne modifie pas le style visuel du bouton (même apparence, juste texte "Envoi...")
  • Inputs du formulaire non désactivés visuellement pendant le chargement
  • Lightbox : pas de placeholder ni skeleton pendant le chargement des images haute résolution
  • CategoryCarousel : aucune indication visuelle que le clic ouvre une autre page (pas de curseur pointer vérifié, pas de label explicatif)
  • Boutons de window controls (—, □, ✕) non fonctionnels mais visuellement identiques aux boutons actifs — risque de confusion

Accessibilité

  • Pas de skip link ("Aller au contenu principal") — première interaction clavier bloquée sur la navigation
  • Boutons décoratifs (—, □, ✕) sans aria-hidden ni disabled — ils apparaissent dans l'arbre d'accessibilité
  • Lightbox : pas de focus trap — Tab sort du modal
  • Lightbox : aucune gestion des touches ArrowLeft / ArrowRight / Escape
  • CategoryCarousel : clic entier sur un <div> — inaccessible au clavier (devrait être un <button> ou <a>)
  • SVG et images décoratives sans aria-hidden="true"
  • Progress bars : aria-valuenow devrait être en pourcentage (0–100), pas le score brut (ex : 3.5)
  • RoleWindow : le texte animé n'est pas annoncé aux screen readers (aria-live manquant)

Mobile UX

  • Modal de détail projet : plein écran sur mobile mais sans gestion du scroll interne — les galeries longues peuvent être inaccessibles
  • Sidebar de /main : scroll horizontal difficile sur mobile, pas de scroll-snap
  • Boutons prev/next du lightbox : non dimensionnés pour les touch targets (minimum recommandé : 44×44px)
  • Pas de swipe gesture sur le lightbox ni sur le carousel

Micro-interactions manquantes

  • Hover state passif sur les tags de projet (aucune interaction)
  • Pas d'animation sur les liens "Voir le projet →" dans /main
  • LogoWindow : le spin au clic mobile est fonctionnel mais rien n'indique à l'utilisateur que le logo est interactif

Parcours utilisateur & conversion

  • Pas de CTA vers /contact depuis la hero — seuls "Mes projets" et "Découvrir" sont présents
  • Page /outils sans lien vers /contact ni vers les projets correspondant aux compétences
  • Pas de message d'appel à l'action en bas de page ou dans un footer
  • Section CTA de /projects/[slug] très minimaliste — 2 boutons simples sans accroche textuelle

UI Interface utilisateur

Cohérence du design system

  • index.css (348 lignes) est un doublon direct de global.css avec les mêmes variables @theme et animations — l'un des deux est legacy
  • Window.tsx : props tailleClassName et tailleTitle remplissent le même rôle — nomenclature incohérente
  • hover:scale-103 dans ProjectCard.tsx est invalide en Tailwind standard (doit être hover:scale-[1.03]) — l'effet hover scale est probablement inactif

Composants manquants ou inconsistants

  • Pas de composant Toast / notification global — chaque composant gère ses propres alertes en inline
  • Pas de composant Skeleton / Loader — les images se chargent sans placeholder
  • Pas de composant Breadcrumb
  • CategoryCarousel est défini mais non intégré dans les pages actuelles (absent de index.astro et main.astro)

États manquants sur les éléments interactifs

  • Boutons de catégorie dans /main et /outils : pas d'état focus-visible différencié du focus global
  • Bouton submit : état loading visuellement identique à l'état idle
  • Liens de navigation dans MenuWindow : état focus non distinct de l'état active
  • Cartes projet dans la grille : pas d'état de focus pour la navigation clavier
  • Boutons prev/next du lightbox : pas d'état active / pressed visible

Images & médias

  • Images de galerie sans alt text individualisé — seul un alt générique ou vide est possible
  • Le champ videoUrl est défini dans le type Project mais aucun composant ne l'affiche — les projets Audiovisuel n'ont pas d'embed vidéo
  • Images lightbox non optimisées pour l'affichage plein écran
  • Portrait hero : même image pour tous les breakpoints, pas de version mobile recadrée

Dark / Light mode

  • Aucun dark mode implémenté
  • prefers-color-scheme: dark non géré — le thème Win95 gris clair est le seul mode disponible

Animations non implémentées ou inutilisées

  • slide-out-right définie mais aucun composant ne l'utilise
  • hover-lift définie mais non appliquée via classe utilitaire
  • chevron-rotate définie mais non appliquée au chevron de la sidebar dans /main
  • La librairie motion (Framer Motion v12) est installée mais aucun composant ne l'utilise — ~50KB ajoutés inutilement au bundle
  • View Transitions : les keyframes slide-left / slide-right sont référencées mais absentes de global.css

Détails visuels perfectibles

  • Grille skill cards dans /outils : passe de 1 à 3 colonnes sans palier md:grid-cols-2
  • Progress bars : même couleur bleue pour tous les niveaux — pas de différenciation visuelle entre "TypeScript: 1/5" et "Figma: 3.5/5"
  • background-attachment: fixed sur l'image de fond — bug iOS connu (parallax cassé sur Safari mobile)
  • Fenêtres sociales dans /contact : couleurs de title bar custom (rose, bleu vif, gris foncé) qui s'écartent du design system Win95 navy/gris

Synthèse exécutive

Le portfolio présente une identité visuelle forte et cohérente (thème Win95), une architecture d'information correcte et des bases d'accessibilité solides. Les blockers immédiats sont : le formulaire de contact non configuré (Web3Forms), le hover scale cassé sur les cartes projet, et les lacunes d'accessibilité clavier dans le lightbox (focus trap + touches). Ces 7 premiers points peuvent être résolus en moins d'une journée de travail et améliorent directement la perception par un recruteur ou un client. Le reste relève d'un polish progressif qui n'affecte pas la navigation principale.
